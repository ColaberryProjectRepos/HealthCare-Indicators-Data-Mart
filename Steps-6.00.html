Prepare Data Warehouse environment in Management Studio 6

1. Create DW environment by downloading SQL Script to create Data Warehouse HEALTHstg, HEALTHods and HEALTHdw DBs:

 

Go to SQL Server Management Studio and use the SQL Script to create DW environment. This will create all the objects, namely, databases, tables, Date Dimension and stored procedures. Run the script below in Management Studio:

·         CreateAllTablesForStgOdsConformedDims.sql

·         deployPopulateDimDate.sql – Creates DimDate for Calendar. Then populate the DimDate using:

o   exec dbo.uspDimCalendarday '12/31/1960', '12/31/2020' for data being extracted

·         CreateAllTablesOdsDwForStaffToPatietRatio.sql

·         CreateAllTablesOdsDwForRoomsTurnover.sql

·         CreateAllTablesOdsDwForTrauma.sql

·         CreateAllTablesOdsDwForMarketing.sql

·         Run the following scripts to create stored procedures for loading Dimension and Fact tables:

o   deployLoadDimensionTablesOnScd1.sql

o   deployLoadDimFactTblsForStaffToPatietRatio.sql

o   deployLoadDimFactTblsForRoomsTurnover.sql

o   deployLoadDimFactTblsForTrauma.sql

o   deployLoadDimFactTblsForMarketing,sql

 

Confirm in Management Studio that the objects have been created by right clicking on each Database tab and selecting Refresh. See below Screen:

 


2.            Create an Excel file, RegionConf.xlsx with 4 worksheets as per the datasets – StaffRatio, Turnover, Trauma and Marketing with 4 columns as shown.  This file will be used to consolidate Region and Country to conformed dimensions. It will capture countries with corresponding regions for each dataset in the 4 worksheets. After identifying which countries and regions are categorized in multiple ways, the file will be sent to the business as a notification to harmonize and agree on a unified categorization and corrected before completing the execution of the packages.
