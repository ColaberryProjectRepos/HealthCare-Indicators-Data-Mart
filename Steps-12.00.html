Load Dimension and Fact Tables in Data Warehouse: 12

 

Double click on the PackageLoadDimAndFactTbls.dtsx. If it’s the first time, you will be presented with a blank pane on the Control Flow Task. 

·         Drag and drop 1 Execute SQL Task from SSIS Toolbox on to the Control Flow Task and name it appropriately as shown in the screenshot below.  Drag and drop a Sequence Container on to the Control Flow Task and name it appropriately as shown and link it with a Precedence Constraint from the first Execute SQL Task.

·         Drag and drop 8 Execute SQL Tasks from SSIS Toolbox into the Sequence Container and group them into 4 sets for the dataset.  Link each set with Precedence Constraint.  Name tasks appropriately as shown in the screenshot below

 



 

·         Double Click in the Execute SQL Task 1 as show in below screenshot and configure to embed the stored procedure for loading the dimension tables based on SCD Type 1. These are DimRegion, DimCountry, DimIndicator, DimGender, DimStaffIncomeGroup, DimMarketingSector, DimTraumaAgeGroup and DimMarketingDemography:

 



 

 

·         Double Click in the first Execute SQL Task marked 2 for the first set of the datasets as shown in below screenshot and configure to embed the stored procedures, uspRemoveKeysStaffDw and uspLoadDimStaff, for loading the dimension table based on SCD Type 2. This is DimStaff for Staff-to-Patient Ratio dataset:

 

 

 


 

·         Double Click in the second Execute SQL Task marked 2 for the first set of the datasets as shown in below screenshot and configure to embed the stored procedure, uspLoadFactStaffDw, for loading the fact table, FactHealthStaff, for Staff-to-Patient Ratio dataset:

 



 

 

·         Repeat configuration marked 2 above for the rest of the Datasets for Turnover Rooms, Trauma and Marketing.
